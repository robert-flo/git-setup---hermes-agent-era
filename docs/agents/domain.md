# Domain Docs

## Antes de explorar

- Lee `CONTEXT.md` en la raíz cuando exista.
- Lee los ADR relevantes en `docs/adr/` cuando existan.

Si estos recursos no existen, continúa sin crear documentos de dominio de forma
preventiva.

## Estructura

Este es un repositorio de contexto único. Cuando sea necesario documentar
decisiones de dominio, usa `CONTEXT.md` y `docs/adr/` en la raíz.
